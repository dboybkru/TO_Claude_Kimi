## Карта объектов

Минимальный полезный вариант:

1. добавить импорт объектов из `objects.csv`;
2. прогнать адреса через геокодер;
3. сохранить `lat/lng`;
4. на странице объектов включить фильтры по типу, региону, статусу ТО и наличию просрочки;
5. для точек без точного адреса показывать приблизительную координату населенного пункта и помечать как `needs_geocode_review`.

Желательные слои карты:

- цвет по состоянию: активный, в ремонте, просрочено ТО;
- форма/иконка по типу: ОС, ОС/ОТС, СКУД;
- группировка маркеров при большом масштабе;
- фильтр "без координат";
- быстрый экспорт выбранных точек в маршрут.

## Планировщик выгодных выездов

Цель: собрать в один рабочий день максимальное число близких точек ТО, не выходя за рабочее время.

Предлагаемая логика:

1. Пользователь выбирает дату, техника, рабочий интервал, старт/финиш, длительность работ на объекте.
2. Система берет объекты, у которых ТО запланировано или просрочено.
3. Точки группируются по близости и району.
4. Для каждой группы считается приблизительный маршрут.
5. В план попадают точки, пока сумма `дорога + работа + запас` помещается в рабочий день.
6. Остальные точки остаются в очереди следующего выезда.

Поля, которые стоит добавить к объекту или настройкам:

- `service_duration_minutes` - типовая длительность ТО на объекте;
- `geocode_status` - `exact`, `approximate`, `failed`, `manual`;
- `geocode_source` - источник координат;
- `route_priority` - ручной приоритет;
- `access_window` - удобные часы доступа, если появятся ограничения.

Первую версию можно сделать без сложной математики: greedy nearest-neighbor с ограничением по времени. Позже заменить расчет дороги на OSRM/GraphHopper/Yandex routing.
